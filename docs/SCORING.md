# Scoring model

lockwarden grades each resolved package **A–F** and rolls the tree up to a worst-grade
summary. Scoring has two layers: a structural layer that works with zero network,
and a known-bad overlay.

For the vectors these signals detect, see [THREAT-MODEL.md](THREAT-MODEL.md).

## Layer 1 — execution-surface score (structural, day-zero capable)

Each analyzer emits *facts* (signals), never severities. The scoring engine maps
`(analyzer, kind)` to a weight. Every signal has an **absolute** weight (the surface
exists) and a **delta** weight (the surface newly appeared in this version vs the
previous one). Delta weights only apply in `--diff` / `--deep` modes; absolute
weights always apply.

| Signal | Absolute | Delta (newly appeared) |
|---|---|---|
| Lifecycle install script | Low–Med | **Critical** |
| `binding.gyp` / node-gyp hook | Low | **Critical** |
| AI-agent hook / MCP manifest | Med | **Critical** |
| IDE task / folder-open file | Med | **High** |
| Main-file size anomaly (>5× vs previous) | — | **High** |
| New transitive dep in a patch release | — | **Critical** |
| Obfuscation markers in install-path files | Med | **High** |
| Phantom dependency (declared, never imported) | Med | — |
| Prebuilt native binary (`.node` file / prebuild fetcher) | Low | **Critical** |

A few delta shapes are elevated to Critical when a compound pattern matches a
validated attack shape (e.g. size-inflation *and* new obfuscation in the same
version — the node-ipc shape; an IDE task that auto-runs on folder open). Each
elevation was verified against the calibration corpus to add zero false Criticals.

## Grades

- Any **Critical** finding on a package → grade **F**.
- Otherwise the numeric sum of finding severities maps to A–F.
- Project rollup = the worst package grade + per-severity counts.

## Layer 2 — known-bad overlay

Resolved `name@version` values are matched against a vendored OSV npm-malware
snapshot and against incident IOC bundles (see [incidents on the
site](https://lockwarden.dev/incidents/)). **Any Layer-2 hit is Critical**,
regardless of the Layer-1 score. This layer is fully offline — the data is bundled
into the package at build time.

## SARIF mapping

`--sarif` emits SARIF 2.1.0 for the GitHub Security tab:

| Grade / severity | SARIF level |
|---|---|
| Critical | `error` |
| High | `warning` |
| Med | `note` |
| Low | suppressed unless `--verbose` |

## Calibration

Weights are **corpus-gated and locked by the full top-500 run (2026-07-06)**. They
are generated by a calibration harness (`corpus/`, not shipped) that runs every
analyzer against the top-500 highest-download benign packages — including their
real version-to-version bumps — and a set of synthetic, defanged malicious fixtures
reproducing each 2026 attack shape. The gate requires clean separation: every
malicious fixture must grade F in delta mode while benign version-bumps produce
zero Criticals (measured: 0 Criticals across 496 real bumps). Changing any weight
requires re-running the gate. See [ARCHITECTURE.md](ARCHITECTURE.md#corpus) for how
the harness works.

## Exit codes

Exit codes are the API — every command is CI-composable:

| Code | Meaning |
|---|---|
| `0` | clean, or findings below `--threshold` |
| `1` | findings at or above `--threshold` (default: `high`) |
| `2` | execution error (bad args, unparseable lockfile, `--offline` violation) |

`--threshold` accepts severity names (`low`/`med`/`high`/`critical`) or grade
letters (`B`/`C`/`D`/`F`).
